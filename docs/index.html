We build the engine. You build the car.

A finished, ready-to-run foundation for AI agents of any complexity. Copy the folder, write a persona, drop in tools — your specialized agent is running.

Get started GitHub →

Philosophy

Race teams don't forge their own engines — they buy a proven one and build the car around it. genesis-agent is that engine for AI agents.

Every agent needs the same heart: model wiring, tool calling, retries, memory, guardrails, a console, deployment. Building it from scratch for every project means weeks of plumbing before any real work. We froze that part, tuned it, and tested it — so the only thing left for you is what makes your agent yours.

⚙ The engine — frozen, ours

  • provider wiring — OpenAI · Anthropic · OpenRouter · offline Ollama
  • automatic tool discovery from tools/*.py
  • the agent loop, retries, usage limits
  • conversation memory + state store (JSON / SQLite)
  • filesystem sandbox, tool policy, approvals
  • live console · HTTP service · Docker · cron

~2k lines of readable Python on Pydantic AI. Small enough to read in an evening.

🏎 The car — yours

  • persona.md — who your agent is
  • tools/*.py — what it can do
  • settings.yaml — how it behaves
  • .env — which model drives it

That's the complete list. A trading desk, a research bot, a support automation — same engine, different car.

Three steps to your agent

01

Copy

One install command — or run the wizard (agent --new) and it scaffolds a fresh agent in a sibling folder, engine included.

02

Describe

Write the role in persona.md. A trader, a researcher, an ops bot — plain markdown, no framework code.

03

Arm

Drop domain tools into tools/ — any documented, type-hinted Python function is auto-discovered. No wiring, no registration.

What's under the hood

Any model, one .env edit

OpenAI, Anthropic, OpenRouter, or fully offline Ollama. Switching providers is a config change, never a refactor.

Any environment

Interactive terminal, headless HTTP service, Docker container, cron / Task Scheduler. Same agent, same behavior.

Safety built in

Filesystem sandbox, tool allow/deny policy, human confirmation for risky calls, per-run usage limits, server auth.

Memory that scales

Conversation memory with auto-compaction, cross-run state store, reflection notes — context stays lean on any model size.

Local-model friendly

Token budgets, tool-output caps, and per-model presets so an 8k-context local model stays sharp, not dumb.

Scales by copy

One folder + one process per agent. 50 agents = 50 folders. No orchestrator, no shared state, no surprises.

Quickstart

Open a terminal in an empty folder and paste — the installer brings uv, dependencies, and .env with it:

# Windows (PowerShell)
> irm https://raw.githubusercontent.com/ysz7/genesis-agent/main/scripts/install.ps1 | iex
# Linux / macOS
$ curl -LsSf https://raw.githubusercontent.com/ysz7/genesis-agent/main/scripts/install.sh | sh

Then set your provider in .env and launch start.cmd / ./start.sh. No API key? Run offline with Ollama.
